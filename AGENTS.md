# AGENTS.md

**Per-attempt history:** each `attempt_N/README.md` is a historical snapshot of the AGENTS.md state under which that attempt ran. It is not read by the agent — it exists only for audit.

**Fitness structure:**
- **Writing this file** (item 0): the meta-fitness governing AGENTS.md itself.
- Operational (items 1–3): workdir, containment, access — preconditions every iteration must respect.
- **Recipe** (item 4): the primary fitness — find the OpenRewrite recipe composition that produces the highest-quality Java 21 conversion of the dataset. May be single-jump or staged across intermediate JDKs.
- **vLLM spin-up** (item 5): the inference endpoint observables, ralph-looped over container / proxy config until satisfied.
- **Dataset rediscovery** (item 6): the corpus the recipe fitness measures against, ralph-looped over candidate repos.
- **Per-failing-repo refinement** (item 7): a finer-grained ralph loop nested under the recipe fitness. Used when the coarse loop plateaus on the build-success metric.
- **Runner saturation** (item 8): keep the verifier host loaded enough to make progress without thrashing.
- **Sub-agent delegation** (item 10): an agent endpoint the orchestrator can hand well-scoped coding subtasks to.

0. **Fitness (writing this file):** keep AGENTS.md compact and outcome-named so the agent re-derives the *how* every iteration from its tools and the corpus.
   - **Constraints:** no implementation instructions the agent can fill itself, no enumerations that age, no justifications for the rule alongside the rule.
   - **Search:** read → why → intent — when revisiting a clause, ask "why is this here?"; if the answer is implementation detail, enumeration, or justification, strip it back to the rule itself.
   - **Reward:** cuts that lose words without losing the rule.
   - **Repeat:** every editing pass.
1. **Workdir:** `$HOME/java_8_11_17_to_java_21`.
2. **Containment:** all build toolchains and recipe execution run inside Docker. If a host resource this project needs is held by something unrelated, free it in favour of this project. Cache external downloads on a host-side bind mount shared across containers, and let the cache survive across iterations.
3. **Access:** SSH calls to the work host share one session, not one per command.
4. **Fitness (recipe):** for every repo in `java21-migration-dataset.json`, find an OpenRewrite recipe SEQUENCE (`{label, jdk, recipes[]}` steps) that builds clean on jv_to. Per-repo, the baseline to beat is the one-shot `UpgradeToJava<jv_to>` recipe. Per-repo search is iterative — start from the default sequenced-Java plan, run, on failure read the actual `[ERROR]` block + maven summary out of the log, ask the LLM proposer for the next mutated chain given the failure signal and prior-attempt history, retry. The corpus is processed as a multi-pass round-robin: each pass gives every still-FAILing repo K more attempts (default K=5), then moves on to the next repo; subsequent passes resume each repo's trajectory from where the previous pass left off; iterate passes until one pass produces zero new PASSes. A proposal identical to a previously-tried chain bails — no value in re-running the same failure. Each step's recipes/versions must be compatible with that step's JDK, and each step's pom/source edits persist into the next step's working tree. Declarative YAML recipes only; chain catalog primitives and pre-built migration recipes. The universal-sequence question (does one chain win across many repos?) is answered AFTER per-repo is healthy, by clustering winning chains and proposing common subsequences — universal is a stretch goal, not a precondition.
    - **Search hints:** the proposer's input must include real failure signal (the `[ERROR]` block + final maven summary), not a header-timestamp tail; logs from successive same-named phases must not accumulate into one file across steps, so use a per-step log dir; common rescuable failure modes — Lombok pinned to a JDK-incompatible version (≤1.18.20 vs J11+, ≤1.18.22 vs J17, ≤1.18.30 vs J21), Spring Boot pinned below its first JDK-supporting release, plugin pins predating the target JDK — each is rescued by a property/dependency bump injected before the JDK-bumping step; prefer stepped framework upgrades over one-shot leaps (~3× more passing stages); pure JDK upgrades are EXEMPT — `UpgradeToJava<N>` chains internally and pure-Java big-leap PASS rate is the same as small-leap; verify the chain under `mvn compile` at every step, not just the last one, so the chain proven to work is the chain that runs; the K=5 slice is small enough that the corpus's 90%-convergence point shows up in observable time and big enough that the proposer can iterate on one root cause before yielding; per-repo trajectory state (history of chains tried + Qwen's last proposal) persists between passes so each new attempt picks up prior hypotheses; record per-attempt wall-clock so post-hoc analysis can answer both attempts-to-pass and time-to-pass.
5. **Fitness (vLLM spin-up):** stand up an OpenAI-compatible chat-completion endpoint that the agent can call from inside Docker containers, serving a model large and tool-capable enough for the recipe loop's prompts, with authentication enforced. Arrive there through a ralph loop over container, model, and reverse-proxy config.
   - **Contract:** thinking mode is supported via `chat_template_kwargs.enable_thinking`; consumers default to thinking enabled — empirically the model with thinking extracts qualitatively richer signal from messy logs (causal chains, suspected root causes, cross-step correlations) than the same model with thinking off, which tends to summarise surface features only; fall back to no-thinking only when thinking output is unparseable.
6. **Fitness (dataset rediscovery):** curate `java21-migration-dataset.json` as distinct-owner lineage samples per (oldest-Java-version × dependency family) cell, where oldest-Java-version ∈ {8, 11, 17} and dependency family ∈ the popular dependencies that OpenRewrite targets as having breaking changes for Java 21 migration. Each entry is a *lineage* — one repo tracked across its Java-version history, with a `commit_sha` recorded at every Java version from the oldest through 21, and each `commit_sha` baseline-buildable inside the runner container under the JDK matching that commit's declared Java version. Iterate candidates through a ralph loop, balancing the matrix.

7. **Fitness (per-failing-repo refinement):** item 4's per-repo iterator IS the refinement loop — no separate stage. This item exists only to call out the inner-loop reward shape distinct from item 4's outer reward: per-repo `0 → 1` flips that survive their own intermediate-step verification, attempts-to-PASS as a tie-breaker. Drop when item 4's iterator is the only refinement mechanism in active use.
8. **Fitness (runner saturation):** keep the verifier host CPU between 60 % and 80 % of cores *while any parent loop is making progress* — this fitness composes with the parent rather than standing alone — saturation only counts toward the composite objective when the parent loop is making progress.
   - **Constraints:** any concurrency dial the agent can reach.
   - **Search:** sample load periodically, decide what to adjust given the recent action history and which parent loop is active.
   - **Reward:** sustained band hit without thrashing or stalling the parent loop.
   - **Repeat:** continuous, dampened against oscillation; pause when no parent loop is active.

10. **Fitness (sub-agent delegation):** stand up an OpenHands (or equivalent) agent endpoint on the work host that the orchestrating agent can hand well-scoped subtasks to, so the orchestrator writes less glue code itself.
    - **Constraints:** sub-agent runs in Docker on the work host, can read/write the project workdir, returns structured results. The sub-agent's sandbox may not reach host-owned docker images or sockets — scope dispatched tasks to ones that don't depend on them.
    - **Search:** ralph loop over container config, model backend wiring, auth.
    - **Reward:** dispatch a representative subtask, get a usable artifact back without writing the script.
    - **Repeat:** every time the orchestrator catches itself writing throwaway scripting glue.
